COMMIT 3018212 / b9accf3 — "fix(ws): accept WebSocket antes de close"
================================================================================
Fecha: 2026-04-14/15 (dos commits con el mismo contenido)

MENSAJE COMPLETO:
----
fix(ws): accept WebSocket antes de close para evitar HTTP 500

Starlette requiere websocket.accept() antes de websocket.close().
Sin esto, el handler lanzaba una excepcion no capturada → HTTP 500.

Ahora: accept primero, luego validar auth/acceso, cerrar si falla.

----

ARCHIVO MODIFICADO:
  backend/collab_ws.py

luego validar, cerrar con codigo de error si la validacion falla.
